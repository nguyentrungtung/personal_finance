# Contract: Investment Ledger — Transaction History & Realized P&L

**Base URL**: `http://localhost:5000/api/v1`
**Date**: 2026-05-14
**Auth**: 🔒 All endpoints require `access_token` httpOnly cookie.

> These endpoints are shared between the Investment Ledger "Trade History" panel (FR-041)
> and the Analytics Realized P&L summary (FR-042).

---

## `GET /api/v1/lots/:id/transactions`

All `asset_transactions` for a specific lot — BUY entry + all SELL segments generated by
FIFO matching. Records are **append-only** and never updated or deleted (Constitution VII).

**Response `data`**:

**Field notes**:

| Field | Notes |
|-------|-------|
| `net_amount` | Negative = cash out (BUY); positive = cash in (SELL, dividend) |
| `realized_pnl` | Non-null on `sell` only; `null` on `buy`, `dividend`, `split` |
| `transaction_type` | `buy \| sell \| dividend \| split` — dividend/split are manual-entry only in v1 |
| `volume` | Signed positive for all types; `"0.0000"` for dividend (income, no unit change) |

**Errors**:
- `401 UNAUTHORIZED`
- `404 NOT_FOUND` — lot `:id` does not exist

- Investment Ledger "Trade History" tab (FR-041)
- Analytics Realized P&L summary (FR-042)

**Query params**:

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `asset_name` | string | — | Filter by ticker / product name |

| `asset_subtype` | string | — | `gold`, `stock`, `mutual_fund`, `apartment`, etc. |

## Notes

- `asset_transactions` records are **append-only** — no UPDATE or DELETE ever (Constitution VII).
  Historical P&L figures never change retroactively.
- A single SELL order that consumes N lots generates N separate rows in `asset_transactions`
  (one per consumed lot segment). The history endpoint returns each segment as a separate row
  with its own `lot_buy_price` and `realized_pnl`.
- `dividend` and `split` records are excluded from this endpoint (SELL-only). They appear in
  `GET /lots/:id/transactions` for individual lot history.
